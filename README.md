# AIOS Dashboard

A public dashboard showing what's automated inside an AI-Operating-System (AIOS) build. The original instance is for [Minimise](https://www.minimise.co.nz/) — every consulting client gets their own fork during their AIOS project.

**Live (Minimise instance):** https://minimise-aios-dashboard.vercel.app

## What it shows

- **Home (`/`)** — total hours saved per week, count of automations live, count of tasks automated, then a department grid and a "what's queued" list.
- **Tasks (`/tasks`)** — every task across the business, ticked when an automation handles it, untick'd when it's still manual.

## How it stays current

Data lives in two JSON files in `public/data/`. They are generated by a sync skill in the operator's separate `*-brain` repo (in Minimise's case, `minimise-brain`'s `.claude/skills/dashboard-sync/`). When that skill runs, it commits fresh `dashboard.json` + `tasks.json` to this repo's `main` branch via the GitHub API, and Vercel rebuilds.

The dashboard repo itself is dumb — it never reads from anywhere else and has no scheduled jobs. All intelligence sits in the brain repo.

## Data contract

Both JSON files conform to a fixed schema. See the spec in the brain repo at `.claude/plans/aios-dashboard-spec.md` (section 1) for the field-by-field definitions.

Quick view: `public/data/dashboard.json` drives the home page, `public/data/tasks.json` drives the tasks page. Each file declares `instance.primary_color`, `instance.text_color`, `instance.background_color`, and `instance.logo_text` so a client fork can rebrand without code changes.

## Fork for a new client (3 steps)

1. **Fork this repo** to the client's GitHub org (or your own, named `<client>-aios-dashboard`).
2. **Edit `public/data/dashboard.json` and `public/data/tasks.json`** — replace `instance.*` with the client's brand tokens (name, primary colour, etc.). Replace `departments` and `coming_next` with the client's data.
3. **Connect to Vercel** as a new project pointing at the fork. No env vars needed.

Setup time: under 30 minutes per client.

For ongoing data updates, give the client (or your operator) the `dashboard-sync` skill template from the brain repo and point it at the new fork. After that, the dashboard updates whenever the operator says "sync the dashboard".

## Stack

- Next.js 16 (App Router) + TypeScript
- TailwindCSS v4
- Vercel
- Plus Jakarta Sans (Google Fonts, OFL — free for commercial use)

## Local dev

```bash
npm install
npm run dev
# open http://localhost:3000
```

Edit JSON files in `public/data/` to preview changes. There is no admin UI — JSON is the interface.
